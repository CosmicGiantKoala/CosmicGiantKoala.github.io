# 포트폴리오 문서 작성 규칙

AI를 통해 Unity 프로젝트의 코드 문서를 자동으로 정리할 때 사용하는 규칙입니다.

---

## 0. 핵심 원칙: 주석은 실제 코드에, 포트폴리오는 가져오기 ⚠️ 필수

**⚠️ 이 원칙은 절대적으로 지켜져야 합니다.**

모든 주석은 **실제 `.cs` 파일에 XML 문서화 주석(`///`) 형식으로 작성**하고, 포트폴리오 문서에서는 해당 주석을 **그대로 가져와 사용**합니다.

**❌ 잘못된 방식:**
- 원본 코드를 수정하지 않고 바로 포트폴리오 문서만 작성
- 포트폴리오 문서에 주석을 직접 작성하고 원본 코드에는 반영하지 않음

**✅ 올바른 방식:**
1. 원본 `.cs` 파일을 열어 XML 주석을 추가 (Phase 1)
2. 주석이 추가된 원본 코드에서 주석을 추출하여 포트폴리오 작성 (Phase 2)

### 2단계 워크플로우 (반드시 순차 실행)

#### 🔴 Phase 1: C# 코드 주석화 (필수 선행 단계)

**이 단계를 완료하지 않으면 절대로 Phase 2로 넘어가지 마세요.**

**Phase 1 완료 기준:**
- [ ] 실제 `.cs` 파일을 열어 **모든 멤버(private 포함)**에 XML 주석 추가
- [ ] 클래스/인터페이스/열거형 선언부에 `<summary>` 주석 추가
- [ ] 모든 메서드(public/private/protected)에 `<summary>`, `<param>`, `<returns>` 주석 추가
- [ ] 모든 필드와 속성에 `<summary>` 주석 추가
- [ ] 주요 로직에 인라인 주석 추가
- [ ] **코드의 모든 요소에 설명이 추가되었는가?**

**Phase 1 완료 후 저장:**
- 수정된 `.cs` 파일을 반드시 저장

#### 🔵 Phase 2: 주석 추출 및 포트폴리오 작성

**Phase 1이 완료된 후에만 실행:**
- 주석이 완벽한 C# 코드에서 내용을 추출
- 추출한 주석을 포트폴리오 문서에 반영
- 코드 스니펫도 동일하게 추출하여 사용

### 자동화 워크플로우 (AI 사용 시) - 반드시 순차 실행

AI를 통해 포트폴리오 작성을 자동화할 때는 **다음 단계를 반드시 순차적으로 실행**합니다:

#### Phase 1: 소스 코드 분석 및 주석 추가 (반드시 먼저 실행)

**⚠️ Phase 1이 완료되기 전에는 Phase 2 작업을 하지 마세요.**

1. **ScriptList.md 확인**: 대상 시스템의 소스 코드 파일 목록 확인
2. **소스 코드 읽기**: 각 `.cs` 파일을 읽어 클래스 구조 파악
3. **XML 주석 추가 (원본 파일 직접 수정)**:
   ```csharp
   // 클래스/인터페이스/열거형 선언부
   /// <summary>
   /// 클래스에 대한 한글 설명
   /// </summary>
   public class ClassName
   
   // 모든 필드(private 포함)
   /// <summary>
   /// 필드에 대한 설명
   /// </summary>
   private int _fieldName;
   
   // 모든 속성
   /// <summary>
   /// 속성에 대한 설명
   /// </summary>
   public int PropertyName { get; set; }
   
   // 모든 메서드
   /// <summary>
   /// 메서드에 대한 설명
   /// </summary>
   /// <param name="paramName">파라미터 설명</param>
   /// <returns>반환값 설명</returns>
   public ReturnType MethodName(ParamType paramName)
   ```
4. **인라인 주석 추가**: 메서드 낸부 비즈니스 로직에 단계별 인라인 주석 추가
5. **원본 파일 저장**: 수정된 `.cs` 파일 저장 (반드시!)

**Phase 1 완료 확인:**
- 원본 코드 파일에 XML 주석이 추가되었는지 확인
- 모든 public/private 멤버에 주석이 있는지 확인
- 파일이 저장되었는지 확인

#### Phase 2: 포트폴리오 문서 작성 (Phase 1 완료 후에만 실행)

**Phase 1에서 추가된 주석을 기반으로 작성:**

1. **Front Matter 작성**: TOML 형식으로 메타데이터 작성 (title, description, weight 등)
2. **섹션 작성**:
   - `## 개요`: 클래스 전체 설명
   - `## 역할`: 주요 책임 목록
   - `## 선언`: C# 선언문 (**Phase 1의 XML 주석 그대로 사용**)
   - `## 멤버`: 이벤트/속성/메서드 (**Phase 1의 XML 주석 그대로 사용**)
   - `## 코드 스니펫`: 주요 로직 스니펫 (클릭스만, **Phase 1의 인라인 주석 포함**)
   - `## 기능 설명`: 기능별 상세 설명
   - `## 의존성/상속 관계`: 상속/구현/의존 클래스
   - `## 사용 예시**: 사용하는 클래스 관점 예시
   - `## 관련 클래스`: 관련 문서 링크

#### Phase 3: 검증 및 완료

1. **체크리스트 확인**: 7절의 체크리스트 항목 모두 확인
2. **원본 코드 주석 확인**: 포트폴리오의 주석이 원본 코드와 일치하는지 확인
3. **링크 검증**: 관련 클래스 링크 형식 확인
4. **최종 저장**: `.md` 파일로 저장

**⚠️ 중요: 각 Phase는 순차적으로 진행되며, 이전 Phase가 완료되어야 다음 Phase로 넘어갑니다.**

**❌ 금지 사항:**
- Phase 1을 걸러뛰고 바로 Phase 2 시작
- 원본 코드를 수정하지 않고 포트폴리오 문서만 작성
- 포트폴리오용 주석을 원본 코드에 반영하지 않음

## 1. Front Matter (메타데이터)

모든 문서는 TOML 형식의 Front Matter로 시작해야 합니다:

```toml
+++
title = "ClassName"
description = "간단한 한 줄 설명"
icon = "code"
date = "2023-05-22T00:27:57+01:00"
lastmod = "2023-05-22T00:27:57+01:00"
draft = false
toc = true
weight = 200
+++
```

### 필수 필드

| 필드 | 설명 | 규칙 |
|------|------|------|
| `title` | 클래스/인터페이스/열거형 이름 | 정확한 C# 식별자 |
| `description` | 한 줄 설명 | 50자 이내, '~하는 클래스/인터페이스/열거형' 형태 |
| `icon` | 아이콘 | `"code"` (고정) |
| `date` | 생성일 | ISO 8601 형식 |
| `lastmod` | 수정일 | ISO 8601 형식 |
| `draft` | 임시 저장 여부 | `false` |
| `toc` | 목차 표시 | `true` |
| `weight` | 정렬 순서 | 사용자가 직접 설정 (문서 간 순서 조정용) |

### Weight 설정 가이드

- 문서 간의 정렬 순서를 조정하기 위한 값입니다
- 필요에 따라 자유롭게 설정 가능합니다
- 일반적으로 100 단위로 구간을 나누어 관리합니다

## 2. 본문 섹션 구조

모든 문서는 다음 섹션을 포함해야 합니다:

```markdown
## 개요
## 역할
## 선언
## 멤버
## 코드 스니펫 (클래스만)
## 기능 설명
## 의존성/상속 관계
## 사용 예시
## 관련 클래스
```

### 2.1 개요

- 클래스/인터페이스/열거형의 전체적인 설명
- 2~3문장으로 작성
- 이 클래스가 속한 시스템과 연결 설명
- 관련 클래스 링크 포함

```markdown
## 개요
`AchievementSystem` 클래스는 SlimeRush 게임의 업적 시스템을 관리하는 핵심 클래스입니다.
이 클래스는 업적의 생성, 상태 관리, 진행 상황 추적, 저장 및 로드 기능을 담당합니다.
```

### 2.2 역할

- 클래스의 주요 책임과 역할을 불릿 포인트로 나열
- 3~5개 항목으로 작성

```markdown
## 역할
- 업적 데이터의 초기화 및 생성
- 업적 상태별 업적 리스트 관리
- 업적 진행 상황 보고 및 처리
- 업적 데이터의 저장 및 로드
```

### 2.3 선언

- C# 선언문을 코드 블록으로 작성
- `public class`, `public interface`, `public enum` 등
- **실제 `.cs` 파일의 XML 주석을 그대로 가져옴**

```markdown
## 선언
```csharp
/// <summary>
/// 업적 시스템을 관리하는 핵심 클래스
/// </summary>

```
```

### 2.5 코드 스니펫 (클릭스만)

**인터페이스와 구조체를 제외한 클래스만 해당.**

- 클래스의 주요 로직을 코드 스니펫으로 추가
- **실제 `.cs` 파일에서 주석과 함께 추출**
- 스니펫이 없는 경우 섹션 생략 가능

```markdown
## 코드 스니펫

### 업적 데이터 초기화
```csharp
/// <summary>
/// 업적 데이터를 생성하고 초기화합니다.
/// SheetData와 SaveData를 병합하여 최종 업적 리스트를 생성합니다.
/// </summary>
private void CreateAchievements()
{
    // SheetData에서 업적 기본 정보 로드
    var sheetData = _achievementInteractor.LoadAchievementData();
    
    // SaveData에서 저장된 진행 상황 로드
    var saveData = _achievementInteractor.LoadAchievementSaveData();
    
    // 병합하여 최종 업적 리스트 생성
    Achievements = _achievementCreator.Create(sheetData, saveData);
    
    // 상태별 리스트 분류
    ClassifyAchievement();
    
    // 이벤트 발생
    OnChangedAchievementState?.Invoke();
}
```
```

### 2.6 기능 설명

- 주요 기능별로 하위 섹션으로 구분

```markdown
## 기능 설명

### 업적 상태 관리 시스템
- **비활성(InActive)**: 선행 조건을 만족하지 못한 업적
- **활성(Active)**: 진행 가능한 업적
- **완료(Complete)**: 모든 조건을 달성한 업적

### 업적 생명주기 관리
1. **초기화**: `Awake()`에서 `CreateAchievements()` 호출
2. **활성화**: 선행 조건 만족 시 자동 활성화
3. **진행**: `Report()`를 통해 진행 상황 갱신
```

## 3. 클래스 유형별 작성 규칙

### 3.1 Interface

- 메서드 중심으로 작성
- 구현하는 클래스들 명시
- 메서드별 `<param>`과 `<returns>` XML 주석 포함
- **코드 스니펫 섹션 없음**

### 3.2 Class

- 이벤트 → 속성 → 메서드 순서로 작성
- `[Inject]` 등의 어트리뷰트 명시
- **의미있는 주요 로직이 있으면 코드 스니펫 섹션 필수**
- 복잡한 로직은 여러 개의 스니펫으로 분리

### 3.3 Enum

- 각 열거형 값에 XML 주석으로 설명
- 값의 의미와 용도 설명
- 관련 사용 예시 포함
- **코드 스니펫 섹션 없음**

### 3.4 Struct

- 속성 중심으로 작성
- 구조체의 용도 설명
- 관련 메서드가 있으면 포함
- **코드 스니펫 섹션 없음**

## 4. 코드 작성 규칙

### 4.1 코드 블록

- 언어: `csharp`
- 들여쓰기: 4 spaces
- 주석: `/// <summary>` XML 문서화 스타일 사용

### 4.2 XML 주석 규칙 (실제 `.cs` 파일용)

```csharp

## 8. 작성 예시 (완성된 예시)

### 실제 C# 코드 (`.cs` 파일)

```csharp
using UnityEngine;
using Zenject;

namespace SlimeRush.Achievement
{
    /// <summary>
    /// 업적 시스템을 관리하는 핵심 클래스
    /// 업적의 생성, 상태 관리, 진행 상황 추적, 저장 및 로드를 담당합니다.
    /// </summary>
    public class AchievementSystem : MonoBehaviour, IAchievementSystem
    {
        #region Events
        
        /// <summary>
        /// 업적 상태가 변경될 때 발생하는 이벤트
        /// </summary>
        public System.Action OnChangedAchievementState;
        
        #endregion
        
        #region Properties
        
        /// <summary>
        /// 모든 업적 리스트
        /// </summary>
        public List<Achievement> Achievements { get; private set; }
        
        /// <summary>
        /// 활성화된 업적 리스트
        /// </summary>
        public List<Achievement> ActiveAchievements { get; private set; }
        
        /// <summary>
        /// 완료된 업적 리스트
        /// </summary>
        public List<Achievement> CompleteAchievements { get; private set; }
        
        /// <summary>
        /// 업적 데이터 인터랙터 (의존성 주입)
        /// </summary>
        [Inject]
        private IAchievementInteractor _achievementInteractor;
        
        /// <summary>
        /// 업적 생성기 (의존성 주입)
        /// </summary>
        [Inject]
        private AchievementCreator _achievementCreator;
        
        #endregion
        
        #region Unity Lifecycle
        
        /// <summary>
        /// 초기화 시 업적 데이터를 생성합니다.
        /// </summary>
        private void Awake()
        {
            CreateAchievements();
        }
        
        #endregion
        
        #region Private Methods
        
        /// <summary>
        /// 업적 데이터를 생성하고 초기화합니다.
        /// SheetData와 SaveData를 병합하여 최종 업적 리스트를 생성합니다.
        /// </summary>
        private void CreateAchievements()
        {
            // SheetData에서 업적 기본 정보 로드
            var sheetData = _achievementInteractor.LoadAchievementData();
            
            // SaveData에서 저장된 진행 상황 로드
            var saveData = _achievementInteractor.LoadAchievementSaveData();
            
            // 병합하여 최종 업적 리스트 생성
            Achievements = _achievementCreator.Create(sheetData, saveData);
            
            // 상태별 리스트 분류
            ClassifyAchievement();
            
            // 이벤트 발생
            OnChangedAchievementState?.Invoke();
        }
        
        /// <summary>
        /// 업적을 상태별로 분류합니다.
        /// </summary>
        private void ClassifyAchievement()
        {
            ActiveAchievements = Achievements.FindAll(a => a.State == AchievementState.Active);
            CompleteAchievements = Achievements.FindAll(a => a.State == AchievementState.Complete);
        }
        
        #endregion
        
        #region Public Methods
        
        /// <summary>
        /// 업적 완료 상태를 확인합니다.
        /// </summary>
        /// <param name="achievementId">확인할 업적 ID</param>
        /// <returns>업적 완료 여부</returns>
        public bool CheckAchievementCompletion(string achievementId)
        {
            var achievement = Achievements.Find(a => a.Id == achievementId);
            return achievement?.State == AchievementState.Complete;
        }
        
        /// <summary>
        /// 업적 진행 상황을 보고합니다.
        /// </summary>
        /// <param name="type">업적 타입</param>
        /// <param name="id">업적 ID 또는 관련 ID</param>
        public void Report(AchievementType type, string id)
        {
            // 해당 타입의 활성 업적 찾기
            var targets = ActiveAchievements.FindAll(a => a.Type == type);
            
            foreach (var achievement in targets)
            {
                // 업적 진행 처리
                achievement.UpdateProgress(id);
                
                // 완료 체크
                if (achievement.IsComplete())
                {
                    CompleteAchievement(achievement);
                }
            }
            
            // 변경사항 저장
            SaveAchievementData();
        }
        
        /// <summary>
        /// 업적을 완료 상태로 변경합니다.
        /// </summary>
        /// <param name="achievement">완료할 업적</param>
        private void CompleteAchievement(Achievement achievement)
        {
            achievement.State = AchievementState.Complete;
            ActiveAchievements.Remove(achievement);
            CompleteAchievements.Add(achievement);
            
            OnChangedAchievementState?.Invoke();
        }
        
        /// <summary>
        /// 업적 데이터를 저장합니다.
        /// </summary>
        private void SaveAchievementData()
        {
            _achievementInteractor.SaveAchievementData(Achievements);
        }
        
        #endregion
    }
}
```
